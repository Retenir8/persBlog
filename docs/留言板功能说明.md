# 留言板功能说明

本文档说明用户主页留言板与全站「留言广场」的用法与权限。

## 功能概览

留言板分两层：

| 场景 | 路径 | 说明 |
|------|------|------|
| 用户主页留言 | `/users/[id]` 页面底部 | 对该用户（主人）留言 |
| 留言广场 | `/guestbook` | 聚合展示各用户主页收到的最新留言 |

留言与文章评论**独立**：留言写在用户主页，评论写在文章详情页。

## 用户主页留言板

### 展示

- 区块标题形如「给 XXX 的留言」。
- 按时间展示留言列表：头像区显示昵称；登录用户留言显示账号名并可点击进入其主页。
- 未登录访客留言可填写「昵称」（`guestName`）。

### 发表留言

- **已登录**：自动以当前账号身份留言，无需填昵称。
- **未登录**：需填写昵称（可选逻辑以实现为准），`authorUserId` 为空。
- 提交：`POST /api/users/[hostUserId]/guestbook`，body `{ content, guestName? }`。

### 删除权限

以下身份可删除某条留言（`canDelete` 逻辑）：

1. 站点 **ADMIN**
2. **留言作者本人**（已登录且 `authorUserId` 为自己）
3. **主页主人**（`hostUserId` 为自己）

删除：`DELETE /api/guestbook/[id]`。

## 留言广场

- 路径：`/guestbook`（导航「留言广场」）。
- **无需登录**即可浏览。
- 展示最近约 100 条全站留言，每条包含：留言者、留言内容、时间、**被留言的主页主人**；点击用户名进入对应用户主页。
- 数据接口：`listRecentGuestbookWall`（`guestbookService`）。

## 数据模型（Prisma）

`GuestbookEntry`：

| 字段 | 说明 |
|------|------|
| `content` | 留言正文 |
| `hostUserId` | 被留言用户（主页主人） |
| `authorUserId` | 留言者用户 ID（访客为空） |
| `guestName` | 访客昵称 |
| `createdAt` | 创建时间 |

## API 摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/users/[id]/guestbook` | 某用户主页留言列表 |
| POST | `/api/users/[id]/guestbook` | 在该用户主页留言 |
| DELETE | `/api/guestbook/[id]` | 删除留言（需权限） |
