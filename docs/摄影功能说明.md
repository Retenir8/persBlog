# 摄影功能说明

本文档说明个人博客「摄影」相册的入口、分类、上传与展示逻辑。

## 功能概览

- **入口**：导航栏「摄影」→ `/photos`。
- **权限**：**仅登录用户**可访问；未登录会重定向到 `/login?callbackUrl=/photos`。
- **范围**：每位用户拥有**私有**相册与分类，互不可见。
- **能力**：创建分类、按分类筛选、上传图片、编辑标题、删除照片、灯箱预览大图。

## 页面结构

1. **页头说明**（`PageIntro`）：标题「摄影」，简述为我的相册与分类。
2. **分类标签**：「全部」+ 各分类名；点击切换 URL 查询参数 `?category=<id>`。
3. **管理区**（登录即具备管理权限）：
   - 新建分类（输入名称后添加）。
   - 上传照片：选择本地图片、可选标题、可选归属分类。
4. **照片网格**：封面图、标题、所属分类；点击可打开灯箱；支持编辑标题、删除。

## 分类

- 分类名在同一用户下唯一（数据库 `@@unique([userId, name])`）。
- 删除分类时，该分类下照片的 `categoryId` 会置空（`onDelete: SetNull`），照片仍保留。
- 分类支持排序字段 `sortOrder`（服务端列表按此与创建时间排序）。

## 上传与存储

- 上传接口：`POST /api/photos`，`multipart/form-data`。
  - 字段：`file`（必填）、`title`（可选）、`categoryId`（可选，空表示未分类）。
- 文件经 `src/lib/photoUpload.ts` 的 `savePhotoFile` 保存到服务器本地（具体目录以实现为准），数据库 `Photo.imageUrl` 存可访问 URL。
- 支持常见图片格式；过大或非法文件由服务端校验并返回错误信息。

## 数据模型（Prisma）

| 模型 | 说明 |
|------|------|
| `PhotoCategory` | 用户相册分类：`name`、`sortOrder`、`userId` |
| `Photo` | 单张照片：`title`、`imageUrl`、`categoryId`、`userId`、`sortOrder` |

## API 摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/photos?category=` | 当前用户照片列表，可选分类筛选 |
| POST | `/api/photos` | 上传照片 |
| PATCH | `/api/photos/[id]` | 更新标题、分类等 |
| DELETE | `/api/photos/[id]` | 删除照片 |
| GET/POST | `/api/photo-categories` | 列表 / 新建分类 |
| PATCH/DELETE | `/api/photo-categories/[id]` | 重命名 / 删除分类 |

以上接口均需登录（`requireAuth`）。
