# persBlog 项目功能总结

## 一、项目定位

persBlog 是一个以个人内容创作为核心、融合社交关系与 AI 陪伴体验的全栈博客系统。项目不只提供传统博客的文章发布、分类、标签、评论和点赞能力，还扩展了好友关系、私信聊天、个人主页装饰、留言板、音乐、书架、摄影相册等模块，形成一个兼具内容展示、社交互动和个性化表达的个人社区型应用。

项目的特色设计主要体现在：

- 全站常驻的 AI 宠物「厨师猫」，支持拖拽、喂食、状态养成、文章读后感和日常问答。
- 围绕关注、互关、好友分组、私信会话、黑名单构建的聊天交友系统。
- 文章访问权限与好友分组联动，实现公开、互关好友、指定好友分组等多层可见性。
- 个人主页组件化展示，支持音乐、心情日历、个性签名、禅语等个性化内容。

## 二、技术栈

### 前端与应用框架

- Next.js 16.2.5，采用 App Router 组织页面与 API。
- React 19.2.4，构建页面、交互组件和全局 UI。
- TypeScript 5，提供类型约束与更稳定的业务开发体验。
- Tailwind CSS 4，负责响应式布局、视觉样式和组件细节。
- Geist 字体，统一项目整体排版风格。

### 后端与数据层

- Next.js Route Handlers，承载 REST API。
- Prisma 5，负责数据库模型定义和类型安全访问。
- PostgreSQL，作为主要业务数据存储。
- Prisma Client 输出到 `src/generated/prisma`，便于项目内统一引用。

### 认证与安全

- NextAuth v5 beta，负责登录会话管理。
- Credentials 登录方式，支持邮箱和密码认证。
- bcryptjs，负责密码哈希与校验。
- JWT 会话策略，降低服务端会话存储依赖。

### 内容编辑与多媒体

- TipTap，提供富文本编辑能力。
- NeteaseCloudMusicApi，支持网易云音乐相关功能。
- sharp 与项目脚本，用于宠物素材等图像处理辅助工作。

### AI 能力

- 通过服务端 `fetch` 调用小米 MiMo 的 OpenAI 兼容聊天接口。
- 支持 `MIMO_API_KEY`、`MIMO_API_BASE`、`MIMO_MODEL` 等环境变量配置。
- AI 能力集中在宠物问答和文章读后感场景中。

## 三、已实现功能总览

### 1. 用户认证与账号系统

- 用户注册、登录、会话保持。
- 用户角色区分，支持普通用户与管理员。
- 用户资料维护，包括头像、简介、签名、位置、职业、GitHub、微信、个人网站等信息。
- 用户个性化设置，包括主页组件、禅语内容、心情日历数据、已读回执设置等。

### 2. 博客文章系统

- 文章创建、编辑、删除、查看。
- 支持文章发布状态。
- 支持分类与标签管理。
- 支持文章浏览量统计。
- 支持文章点赞。
- 支持文章全文搜索。
- 支持我的文章列表。
- 支持首页和文章列表的信息流展示。
- 支持文章详情页互动组件。

### 3. 评论系统

- 支持文章评论。
- 支持游客昵称评论。
- 支持登录用户评论。
- 支持嵌套回复结构。
- 支持评论删除。

### 4. 文章可见性与好友分组

- 文章默认支持公开可见。
- 支持互关好友可见。
- 支持按好友分组控制文章可见范围。
- 支持可见分组和不可见分组组合配置。
- 文章阅读权限会在服务端进行校验，AI 宠物读取文章生成读后感时同样遵循权限控制。

### 5. 社交关系系统

- 支持关注与取消关注。
- 支持查看互关好友。
- 支持关注请求相关接口。
- 支持好友分组管理。
- 支持将好友加入不同分组。
- 支持基于好友关系的订阅信息流。
- 支持用户主页上的关系操作。

### 6. 聊天交友系统

项目已经实现站内私信与好友互动能力，主要包括：

- 一对一会话模型。
- 消息发送与接收。
- 会话列表。
- 会话详情页。
- 好友侧栏与移动端适配布局。
- 消息文本内容。
- 文件类消息上传，上传内容保存到 `public/uploads`。
- 消息已读状态。
- 会话置顶。
- 会话静音。
- 导航栏未读消息角标。
- 2 分钟内消息撤回。
- 黑名单模型与管理接口。

当前聊天系统以 REST API 和前端请求拉取为主，尚未引入 WebSocket、Socket.IO 或 SSE 等实时通信通道，因此更适合描述为站内私信系统，而不是强实时 IM 系统。

### 7. AI 宠物系统

AI 宠物是项目最具辨识度的特色功能之一。系统中的宠物为全站常驻的「厨师猫」，具有陪伴、互动、问答和文章阅读辅助能力。

已实现能力包括：

- 全站挂载，进入任意页面都能看到宠物。
- 支持拖拽定位，用户可以将宠物移动到合适位置。
- 支持点击互动，触发跳跃、高兴等动画反馈。
- 支持右键菜单，集中入口包括喂食、问答等操作。
- 支持本地养成数值，包括饱食度和心情。
- 宠物状态会随时间自然衰减。
- 支持喂食，不同食物提升宠物状态。
- 支持日常提问，用户可以直接与宠物聊天。
- 支持文章读后感，登录用户可让宠物阅读当前文章并生成简短反馈。
- 文章读后感会经过服务端文章权限校验，避免越权读取不可见文章。
- 服务端会将文章 HTML 内容转换为纯文本并截断，控制传给 AI 的上下文长度。
- MiMo 接口具备默认域名和备选域名容错设计。

## 四、核心特色设计

### AI 宠物：把博客从工具变成陪伴式空间

传统博客通常只关注内容管理，而 persBlog 将 AI 宠物放在全站布局中，让它成为用户浏览、写作、阅读时的陪伴角色。宠物不只是装饰物，它具备状态养成、喂食、互动动画和 AI 对话能力，还能结合文章内容生成读后感。

这一设计让博客具备更强的情绪价值：

- 写作者可以获得轻量反馈。
- 读者可以通过宠物理解文章内容。
- 用户能在浏览过程中感受到持续陪伴。
- AI 能力被包装成角色体验，而不是生硬的工具按钮。

### 聊天交友系统：从内容社区延伸到关系网络

项目围绕关注、互关、好友分组和私信会话构建社交关系。用户不仅能阅读文章，也能关注作者、沉淀好友关系、管理分组，并通过站内私信交流。

亮点包括：

- 互关好友作为更可信的社交关系。
- 好友分组直接参与文章可见性控制。
- 私信支持置顶、静音、已读、撤回等产品细节。
- 黑名单机制为后续社交安全提供基础。

### 分组可见性：内容权限与社交关系结合

文章可见性不只是简单的公开或私密，而是与好友分组系统结合。用户可以将不同好友放入不同分组，再决定文章面向哪些关系开放。这使项目更接近真实社交平台的内容分发逻辑。

### 多内容形态：博客、音乐、书架、摄影并存

persBlog 不局限于文字内容，还支持音乐收藏、读书记录和摄影展示。它更像一个数字个人主页，能展示用户的兴趣、生活、审美与创作。

## 六、项目总结

persBlog 已经从单纯的博客系统扩展为一个完整的个人数字空间。它以文章创作为核心，通过 AI 宠物增强陪伴感，通过聊天交友系统增强人与人的连接，通过音乐、书架、摄影和主页组件增强个人表达。

当前项目的核心竞争力可以概括为：

- 内容创作完整：文章、分类、标签、评论、点赞、搜索、浏览量均已覆盖。
- 社交关系清晰：关注、互关、好友分组、订阅、私信、黑名单形成闭环。
- AI 体验有角色感：AI 宠物不是独立工具，而是融入全站体验的陪伴角色。
- 个性化表达丰富：主页组件、多媒体模块和主题系统增强个人空间属性。
- 数据模型扎实：Prisma schema 已覆盖用户、文章、评论、社交、聊天、多媒体和个性化数据。

后续如果继续迭代，可以重点考虑实时聊天通道、AI 宠物长期记忆、宠物等级系统、消息通知推送、更多主页组件市场化配置等方向。
