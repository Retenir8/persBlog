# 主题更换说明

本文档说明博客「框体主题」的使用方式、默认行为，以及开发者如何扩展或修改主题系统。

## 功能概览

| 能力 | 说明 |
|------|------|
| 框体配色 | 自定义卡片、面板、评论区等**框体底色**的主色调 |
| 多级饱和度 | 同一主题色下，顶层框体最饱和，内层逐级变淡 |
| 本机保存 | 颜色写入浏览器 `localStorage`，换设备需重新设置 |
| 深浅色跟随系统 | 未自定义时随 `prefers-color-scheme`；自定义后在深浅色下自动混色 |
| 一键恢复 | 恢复为纯黑白默认，不使用灰绿色调 |

**说明：** 当前主题主要作用于「框体 / 面板」层，**不会**改变全站正文文字色、导航栏白底、普通按钮与输入框的硬编码样式（搜索栏输入框除外）。

## 访客 / 作者：如何更换主题

### 打开调色板

1. 在任意页面右侧中间，点击 **调色板图标**（圆形按钮）。
2. 面板从右侧向左展开，标题为「框体底色」。

### 选色方式

- **色板**：在二维区域拖动，调节饱和度与明度。
- **色相条**：拖动彩虹条改变色相。
- **RGB / HEX**：手动输入数值。
- 拖拽过程中页面会**实时预览**；松手后颜色才会写入本机并持久化。

### 恢复默认

- 使用自定义颜色后，调色板下方会出现 **重置** 按钮（循环箭头图标）。
- 点击后恢复为：
  - **浅色模式**：白底 `#ffffff`、黑字、黑边框（半透明）
  - **深色模式**：黑底、白字、白边框（半透明）

### 数据存储

- 存储键名：`persblog-surface-theme`
- 存储位置：浏览器 `localStorage`
- 格式示例：

```json
{
  "rgb": { "r": 200, "g": 230, "b": 210 },
  "isCustom": true
}
```

清除站点数据或手动删除该键即可取消自定义主题。

## 主题作用范围

### 会随主题变化的区域

使用 `src/lib/surfaceStyles.ts` 中样式类的组件，例如：

| 样式类 | 典型用途 |
|--------|----------|
| `surfacePanelTopClass` | 页面介绍区、文章正文外壳、登录框 |
| `surfacePanelClass` | 评论区外壳、书架管理区 |
| `surfacePanelNestedClass` | 文章卡片、留言项、草稿列表项 |
| `surfacePanelSubtleClass` | 搜索栏外框、分类/标签管理区 |
| `surfaceFieldClass` | 文章列表搜索输入框 |
| `surfaceChipClass` | 文章卡片上的分类、标签胶囊 |
| `postCardSurfaceClass` | 文章列表卡片 |

### 不会随框体主题变化的区域

- 导航栏（固定 `bg-white/80` 等）
- 普通 `Button`、`Input`（写文章表单、书评输入等）
- 页脚
- 宠物、音乐播放器等独立 UI

若需让新页面参与主题，请使用上述 `surface*` 类，而不是写死 `bg-white`。

## 多级框体原理

用户只需选一种 **强调色**（`--surface-accent`），其余层级由 CSS 自动派生：

```
顶层 (surface-0) ── 饱和度最高，如 PageIntro、文章正文区
    └─ 中层 (surface-1) ── 评论区外壳
        └─ 内层 (surface-2) ── 卡片、留言项
            └─ 淡层 (surface-3) ── 搜索区、分类管理
                └─ 胶囊 (surface-chip) ── 分类/标签标签
```

自定义模式下，`<html>` 会带上 `data-surface-custom`，`globals.css` 内用 `color-mix()` 将 `--surface-accent` 与页面底色混合，生成各级 `--surface-*-bg` 与边框色。

## 开发者指南

### 相关文件

| 文件 | 职责 |
|------|------|
| `src/lib/theme/colorUtils.ts` | RGB / HSV / HEX 转换 |
| `src/lib/theme/surfaceTheme.ts` | 应用主题、读写 `localStorage`、首屏 bootstrap 脚本 |
| `src/components/theme/ThemeProvider.tsx` | React Context：`setColor`、`reset` |
| `src/components/theme/ColorPickerPanel.tsx` | 右侧调色板 UI |
| `src/components/providers.tsx` | 挂载 `ThemeProvider` 与 `ColorPickerPanel` |
| `src/app/layout.tsx` | 注入 bootstrap 脚本，避免首屏闪烁 |
| `src/app/globals.css` | CSS 变量默认值与 `data-surface-custom` 派生规则 |
| `src/lib/surfaceStyles.ts` | 对外暴露的 Tailwind 样式类 |

### 数据流

```
用户选色
  → ColorPickerPanel.previewHsv（拖拽时仅预览）
  → ColorPickerPanel.commitColor / ThemeProvider.setColor（松手或输入框提交）
  → applySurfaceTheme + saveSurfaceTheme
  → document.documentElement 设置 --surface-accent 与 data-surface-custom
```

首屏加载时：

1. `layout.tsx` 内联 `SURFACE_THEME_BOOTSTRAP_SCRIPT`，在 React 水合前从 `localStorage` 恢复 accent。
2. `ThemeProvider` 的 `useEffect` 再次读取并同步 React 状态。

### 在新组件中使用主题

```tsx
import { surfacePanelNestedClass } from "@/lib/surfaceStyles";

export function MyCard({ children }: { children: React.ReactNode }) {
  return (
    <section className={`p-4 ${surfacePanelNestedClass}`}>
      {children}
    </section>
  );
}
```

在客户端组件中读取当前颜色：

```tsx
"use client";
import { useSurfaceTheme } from "@/components/theme/ThemeProvider";

const { rgb, isCustom, setColor, reset } = useSurfaceTheme();
```

### 修改默认主题（代码层）

编辑 `src/lib/theme/surfaceTheme.ts`：

```ts
export const DEFAULT_SURFACE_THEME: SurfaceThemeState = {
  rgb: { r: 255, g: 255, b: 255 },
  isCustom: false,
};
```

编辑 `src/app/globals.css` 中 `:root` 与 `@media (prefers-color-scheme: dark)` 下的 `--background`、`--foreground`、`--surface-*` 等变量，可调整**未自定义**时的黑白默认与各层级表现。

### 调整多级饱和度比例

在 `globals.css` 的 `:root[data-surface-custom]` 中修改 `color-mix` 百分比，例如：

```css
--surface-1-bg: color-mix(in srgb, var(--surface-accent) 50%, #ffffff 50%);
--surface-2-bg: color-mix(in srgb, var(--surface-accent) 28%, #ffffff 72%);
```

数值越大，该层越接近用户所选强调色。

### 修改存储键名

若需更换 `localStorage` 键，请同时修改：

1. `SURFACE_THEME_STORAGE_KEY`（`surfaceTheme.ts`）
2. `SURFACE_THEME_BOOTSTRAP_SCRIPT` 中的键名（同文件，由常量序列化生成）

否则旧用户本地数据将无法被读取。

### 常见问题

**Q：刷新后颜色闪一下？**  
A：确认 `layout.tsx` 中 bootstrap 脚本在 `<body>` 最前执行，且 `<html>` 带有 `suppressHydrationWarning`（脚本会提前修改 `documentElement` 属性）。

**Q：某块区域不变色？**  
A：检查是否使用了 `bg-white` 等硬编码类；改为 `surfacePanel*` / `surfaceChip*` 类。

**Q：拖拽调色板报错？**  
A：拖拽期间只更新本地预览，松手后才调用 `setColor`；若仍异常，检查是否在 `useEffect` 中根据 `rgb` 循环触发 `setColor`。

## 与系统深色模式的关系

| 状态 | 行为 |
|------|------|
| 未自定义（`isCustom: false`） | 浅色为白底黑字；深色为黑底白字，由 `prefers-color-scheme` 控制 |
| 已自定义（`data-surface-custom`） | 框体各级用 accent 与页面底色混色；`--background` / `--foreground` 仍随系统深浅色切换 |

因此：自定义主题主要改变**卡片框体**色调，页面大背景与文字仍跟随系统深浅色规则。
