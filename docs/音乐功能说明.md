# 音乐功能说明

本文档说明「音乐」页如何收藏网易云单曲/歌单、站内播放与外链打开。

## 功能概览

- **入口**：导航栏「音乐」→ `/music`。
- **权限**：**仅登录用户**可访问与管理；列表为**用户私有**。
- **来源**：通过粘贴网易云**网页版**分享链接添加（单曲或歌单）。
- **展示**：卡片网格，含封面、歌名/备注、副标题（艺术家或「网易云歌单」）。
- **播放**：
  - **单曲（SONG）**：站内 `SongAudioPlayer` 拉取播放地址并播放。
  - **歌单（PLAYLIST）**：嵌入网易云官方 outchain 播放器 iframe。
- **外链**：每张卡片下方「用网易云打开」，新标签页跳转网易云网页。

## 添加音乐

在页面顶部「添加网易云音乐」区域：

1. **链接**：粘贴如 `https://music.163.com/song?id=…` 或歌单链接（解析逻辑见 `src/lib/netease.ts` 的 `parseNeteaseUrl`）。
2. **歌名**：必填，作为列表与卡片上的显示名称（存入 `note` 字段）。
3. 点击「添加」→ `POST /api/music`。

添加成功后服务端会尝试拉取封面、曲名、歌手等元数据（`fetchNeteaseDisplayMeta`），并写入 `MusicItem`。

同一用户对同一 `neteaseId` + `kind` 不可重复添加（唯一约束，冲突返回 409）。

## 播放技术说明

### 单曲播放地址

- 客户端请求：`GET /api/music/play-url?songId=<网易云数字ID>`。
- 服务端通过 `NeteaseCloudMusicApi` 解析真实音频 URL。
- 若为 HTTP 链接，会改写为同源代理：`/api/music/audioProxy?src=…`（避免 HTTPS 页面混合内容无法播放）。

### 歌单嵌入

- iframe 地址由 `neteasePlayerEmbedSrc(kind, neteaseId)` 生成（网易云 outchain player）。

### 网易云网页链接

- `neteaseWebUrl(kind, neteaseId)` → `https://music.163.com/#/song?id=…` 或 `#/playlist?id=…`。

## 可选环境配置

若解析或播放失败，可检查：

| 变量 | 说明 |
|------|------|
| 网易云 Cookie | 见 `src/lib/neteaseCookie.ts`（部分接口需登录态 Cookie 才能拿到播放链） |

`next.config.ts` 中将 `NeteaseCloudMusicApi` 列为 `serverExternalPackages`，避免打包问题。

## 数据模型（Prisma）

`MusicItem` 字段要点：

| 字段 | 说明 |
|------|------|
| `neteaseId` | 网易云资源 ID |
| `kind` | `SONG` \| `PLAYLIST` |
| `note` | 用户填写的展示名 |
| `coverUrl` / `trackTitle` / `artistName` | 自动拉取的元数据 |
| `userId` | 所属用户 |

## API 摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/music` | 当前用户音乐列表 |
| POST | `/api/music` | 添加（body: `url`, `note`） |
| DELETE | `/api/music/[id]` | 从列表移除 |
| GET | `/api/music/play-url?songId=` | 解析单曲播放 URL |
| GET | `/api/music/audioProxy?src=` | 音频流同源代理 |

## 与小组件的关系

在「我的博客」启用 **音乐播放器** 小组件时，会读取音乐列表中**第一首单曲（SONG）** 用于迷你播放。详见 `docs/小组件功能说明.md`。
