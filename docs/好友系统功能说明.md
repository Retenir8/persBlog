# 好友系统功能说明

本文档说明博客「好友系统」的各项功能，便于使用与维护。

## 功能概览

好友系统包含以下核心功能：
- **好友分组管理**：创建、编辑、删除好友分组
- **文章可见性控制**：按好友分组设置文章可见范围
- **私信功能**：用户间一对一聊天
- **消息撤回**：发送后1分钟内可撤回
- **消息时间戳**：所有消息显示发送时间
- **右键菜单**：对用户消息进行免打扰、置顶、拉黑操作

## 好友分组管理

### 入口
- 导航栏「个人信息」→「好友分组」页面
- URL：`/users/[id]/friends`

### 功能说明
| 功能 | 说明 |
|------|------|
| 创建分组 | 输入分组名称，创建新的好友分组 |
| 默认分组 | 第一个分组自动设为默认分组 |
| 添加成员 | 从已有用户列表中添加好友到分组 |
| 移除成员 | 从分组中移除用户 |
| 删除分组 | 删除整个分组（成员不会被删除） |
| 设置默认 | 指定某分组为默认分组 |

### 数据模型（Prisma）

```
FriendGroup {
  id          String              @id @default(cuid())
  name        String              // 分组名称
  userId      String              // 所属用户
  isDefault   Boolean             // 是否默认分组
  sortOrder   Int                 // 排序
  members     FriendGroupMember[]
}

FriendGroupMember {
  id              String       @id @default(cuid())
  groupId         String       // 所属分组
  userId          String       // 用户ID
  addedAt         DateTime     // 添加时间
  lastInteractedAt DateTime    // 最后互动时间
}
```

### API 摘要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/friend-groups` | 获取当前用户的所有分组 |
| POST | `/api/friend-groups` | 创建新分组 |
| PATCH | `/api/friend-groups/[id]` | 更新分组信息 |
| DELETE | `/api/friend-groups/[id]` | 删除分组 |

## 文章可见性控制

### 功能说明
发布或编辑文章时，可以设置文章的可见范围：

| 可见性选项 | 说明 |
|-----------|------|
| 公开 (PUBLIC) | 所有人均可查看 |
| 好友可见 (FRIENDS) | 所有好友可见 |
| 分组可见 (GROUP) | 指定分组的好友可见 |
| 分组不可见 | 指定分组的好友不可见 |

### 使用方式

1. 在文章编辑器（`/posts/new` 或 `/posts/[id]/edit`）中
2. 找到「可见性设置」区域
3. 选择可见性类型：
   - 公开：所有人均可查看
   - 仅好友：所有好友可见
   - 指定分组可见：勾选允许查看的分组
   - 指定分组不可见：勾选禁止查看的分组

### 数据存储

```
Post {
  visibility         String    @default("PUBLIC")
  visibleGroupIds   String?   @default("[]")   // 允许查看的分组ID列表（JSON数组）
  invisibleGroupIds String?   @default("[]")   // 禁止查看的分组ID列表（JSON数组）
}
```

### 查看权限逻辑

查看文章时的权限检查由 `canViewPost` 函数实现：

```typescript
async function canViewPost(post, viewerId, viewerGroups) {
  // 公开文章任何人都可查看
  if (post.visibility === 'PUBLIC') return true;

  // 非公开文章需要登录
  if (!viewerId) return false;

  // 作者本人始终可以查看
  if (post.authorId === viewerId) return true;

  // 仅好友可见：检查是否有好友关系
  if (post.visibility === 'FRIENDS') {
    return await checkFriendship(post.authorId, viewerId);
  }

  // 分组可见：检查是否在允许的分组中
  if (post.visibility === 'GROUP') {
    const allowedGroups = JSON.parse(post.visibleGroupIds || '[]');
    return viewerGroups.some(g => allowedGroups.includes(g.id));
  }

  return false;
}
```

| 方法 | 路径 | 说明 |

| GET | `/api/posts/[id]` | 获取单篇文章（包含可见性检查） |
| POST | `/api/posts` | 创建文章（支持可见性参数） |
| PATCH | `/api/posts/[id]` | 更新文章（包括可见性设置） |

## 消息撤回功能

### 功能说明
- **撤回时限**：消息发送后 **1分钟内** 可撤回
- **显示条件**：仅发送者本人可见撤回按钮
- **交互方式**：右键点击可撤回的消息 → 选择「撤回」

### 撤回逻辑

```typescript
function canRetractMessage(message, currentUserId) {
  const now = new Date();
  const sentTime = new Date(message.createdAt);
  const oneMinute = 60 * 1000;

  // 检查是否是发送者
  if (message.senderId !== currentUserId) return false;

  // 检查是否在1分钟内
  if (now.getTime() - sentTime.getTime() > oneMinute) return false;

  // 检查是否已被撤回
  if (message.isRetracted) return false;

  return true;
}
```

## 消息时间戳

### 功能说明
- 每条消息显示发送时间
- 时间格式：根据发送时间自动选择
  - 今天：显示具体时间（如 14:30）
  - 昨天：显示「昨天 14:30」
  - 更早：显示日期（如 2024-05-15 14:30）

### 显示效果

```
[今天 14:30] 用户A: 你好
[昨天 10:15] 用户B: 在吗？
[2024-05-15 09:00] 用户A: 有空聊聊吗
```

## 右键菜单功能

### 功能说明
在消息页面右键点击用户名称或头像，显示操作菜单：

| 功能 | 说明 |
|------|------|
| 免打扰 | 开启后该对话不再推送通知 |
| 置顶 | 将对话固定在列表顶部 |
| 拉黑 | 将用户加入黑名单，后续无法发送消息 |

### API 说明

#### 设置免打扰
```
PATCH /api/conversations/[id]/mute
Body: { "isMuted": true/false }
```

#### 设置置顶
```
PATCH /api/conversations/[id]/pin
Body: { "isPinned": true/false }
```

#### 拉黑用户
```
POST /api/blacklist
Body: { "userId": "目标用户ID" }
```

#### 取消拉黑
```
DELETE /api/blacklist/[userId]
```

## 维护注意

### 数据库同步
修改 `schema.prisma` 后需执行：
```bash
npx prisma generate    # 生成 Prisma Client
npx prisma db push     # 同步数据库结构
```

### 权限检查
所有写操作（发送消息、撤回、设置等）均需登录验证。

### 可见性缓存
文章可见性检查会查询好友关系和分组信息，确保在添加/删除好友后，权限检查结果正确更新。
