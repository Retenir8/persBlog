# 小组件功能说明

本文档说明个人博客中「小组件」相关的设计、数据与代码位置，便于后续维护与扩展。

## 功能概览

- **入口**：导航栏「**小组件**」按钮，点击展开下拉菜单（与「文章」菜单交互一致）。
- **预览与添加**：菜单项对应各小组件名称，链到 `/widgets/preview/[key]`；可预览后**添加到我的博客**（`/myposts`）。
- **展示位置**：已启用的小组件显示在登录后的「**我的博客**」页（`/myposts`），**一言卡片**单独占满宽；**数字时钟、心情日历、音乐播放器**在同一行三列，各占约 1/3 宽度，并与心情日历区域统一最小高度。

## 下拉菜单顺序

导航下拉中的顺序由 `WIDGET_REGISTRY` 数组定义，当前为：

1. 数字时钟  
2. 心情日历  
3. 一言卡片  
4. 音乐播放器（置于最后一项）

三列博客页上的列顺序由 `WIDGET_TRIO_KEYS` 单独定义（时钟 | 心情日历 | 音乐），与下拉顺序无关。

## 已支持的小组件

| key | 名称 | 说明摘要 |
|-----|------|----------|
| `digital-clock` | 数字时钟 | 实时日期与时间；三列布局下为紧凑样式，字号相对加大。 |
| `mood-calendar` | 心情日历 | 月历格子点击切换背景色表示心情；下方有颜色图例。 |
| `zen-quote` | 一言卡片 | 支持自定义文案；留空则使用内置默认句。 |
| `music-player` | 音乐播放器 | 播放「音乐」页收藏列表中的第一首单曲（`SONG`）。 |

注册与校验逻辑：`src/lib/widgets/registry.ts`（`WIDGET_REGISTRY`、`WIDGET_TRIO_KEYS`、`isKnownWidgetKey`、`getWidgetByKey`）。

## 心情日历颜色语义

存储值为英文键：`angry`、`sad`、`calm`、`happy`（及未选日期的空状态）。点击同一日期按顺序循环：**无色 → 愤怒(红) → 悲伤(蓝) → 平静(绿) → 开心(黄) → 无色**。

样式与循环定义：`src/lib/widgets/moodCalendar.ts`（`MOOD_CYCLE`、`MOOD_CELL_BG`、`nextMoodKey`、`parseMoodCalendarData`、`sanitizeMoodCalendarPayload`）。

## 数据模型（Prisma）

`User` 表上与小组件相关的字段（见 `prisma/schema.prisma`）：

- `profileWidgets`：`String[]`，已启用的小组件 key 列表，顺序即展示顺序。  
- `zenQuoteText`：`String?`，一言卡片自定义内容。  
- `moodCalendarData`：`Json`，心情日历 `{ "YYYY-MM-DD": "angry"|"sad"|"calm"|"happy" }`。

迁移文件位于 `prisma/migrations/` 下（如 `add_profile_widgets`、`add_zen_quote_text`、`add_mood_calendar_data`）。

## API 与读写说明

- **PATCH** `/api/users/[id]`：在已登录且为本人时，可更新资料字段及上述小组件字段。  
- **小组件字段写入**：`profileWidgets`、`zenQuoteText`、`moodCalendarData` 通过 `src/lib/services/userWidgetSql.ts` 中的 **`$executeRaw` / `$executeRawUnsafe`** 更新，避免在本地 Prisma Client 未执行 `prisma generate`、与 schema 不同步时出现 `Unknown field` 校验错误。  
- **读取**：`/myposts` 与小组件预览页通过 **`getUserWidgetFields`**（`$queryRaw`）读取三列，同样绕过过时 Client 的 `select` 限制。

更新载荷拼装：`src/app/api/users/[id]/route.ts`（`buildUserUpdatePayload` + 拆分 widget 字段后写库）。

## 本地开发注意

1. 数据库迁移：`npx prisma migrate deploy`  
2. 若修改了 `schema.prisma`，建议在**停止 `npm run dev`** 后执行 `npx prisma generate`，以免 Windows 下引擎文件占用导致 EPERM。  
3. 在 Client 未成功生成时，小组件列仍依赖 Raw SQL 路径，功能可正常使用。

## 扩展新小组件（简要）

1. 在 `prisma/schema.prisma` 的 `User` 上增加所需字段（或使用 JSON 扩展），并编写迁移。  
2. 在 `WIDGET_REGISTRY` 中追加一项（若希望出现在三列行，再考虑是否加入 `WIDGET_TRIO_KEYS` 与布局）。  
3. 在 `WidgetRenderer` 中增加 `case` 分支并实现组件。  
4. 若新字段需绕过 Prisma Client 校验，在 `userWidgetSql.ts` 的查询与更新中补充对应列。  
5. 预览路由 `/widgets/preview/[key]` 会随 `getWidgetByKey` 自动支持新 key。
