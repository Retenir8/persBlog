# 宠物 AI 功能说明

本文档说明全站右下角「厨师猫」虚拟宠物的交互、AI 能力与喂食系统，便于使用与维护。

## 功能概览

厨师猫为**全局悬浮组件**，在任意页面右下角展示，主要能力包括：

| 能力 | 说明 |
|------|------|
| 左键互动 | 空闲时点击随机播放跳跃或高兴摇头动画；可拖拽改变位置 |
| 右键菜单 | 「给猫咪喂食」「向猫咪提问」 |
| 登录引导 | 登录成功后一次性气泡提示操作方式 |
| 文章读文 | 文章详情页「让猫咪读一读」，由 AI 生成读后感（需登录） |
| 自由提问 | 任意页面通过右键菜单向猫咪提问（MiMo 模型） |
| 喂食养成 | 查看饱食度 / 心情值，点击食物恢复；数值随时间缓慢下降 |

## 交互说明

### 左键

- **拖拽**：按住左键移动可调整猫咪在视口中的位置（相对右下角偏移）。
- **点击**（未拖拽且处于 idle）：约 50% 概率跳跃，约 50% 概率播放透明摇头 GIF。

### 右键菜单

1. **给猫咪喂食**  
   打开喂食面板，展示饱食度、心情值进度条（0～100）及 8 种食物。点击食物即可喂食，喂食成功后猫咪会播放摇头动画。数据保存在浏览器 `localStorage`（键名 `persblog-chef-cat-vitals-v1`），换设备或清缓存会重置。

2. **向猫咪提问**  
   打开对话框，输入问题后由小米 MiMo 生成简短、口语化的中文回答。

### 登录后引导

从本站登录表单成功登录后，跳转落地页会显示一次气泡：「左键点击我与我互动，右键点击我有更多功能哦。」约 12 秒后自动关闭，也可手动关闭。

## 文章「让猫咪读一读」

- **入口**：文章详情页顶栏，「返回列表」右侧。
- **权限**：**必须登录**；未登录显示「登录后让猫咪读一读」，跳转登录页并带回跳地址。
- **行为**：点击后请求 `POST /api/pet/mimo`，请求体为 `{ postId }`；服务端按与详情页相同的权限拉取正文（HTML 转纯文本，过长会截断），返回猫咪口吻的读后感。
- **面板**：标题「猫咪读文」，支持关闭、再读一次。

## AI 接口（`/api/pet/mimo`）

统一走小米 MiMo OpenAI 兼容接口，需在 `.env` 配置：

| 变量 | 说明 |
|------|------|
| `MIMO_API_KEY` | 必填，否则返回 503 |
| `MIMO_MODEL` | 可选，默认 `mimo-v2-pro` |
| `MIMO_API_BASE` | 可选，未设置时依次尝试官方两个默认地址 |

### 请求模式

| 模式 | 请求体 | 说明 |
|------|--------|------|
| 自由提问 | `{ "question": "..." }` | 最长 4000 字 |
| 读文 | `{ "postId": "..." }` | 需登录；不可与 `question` 同时传 |

## 喂食系统

### 饱食度与心情

- 初始约：饱食 72、心情 68（首次写入本地时）。
- 随时间衰减（按上次结算时间累计小时数）：饱食约 **2.2 / 小时**，心情约 **2.8 / 小时**（向下取整，不低于 0）。
- 喂食时在现有值上累加，上限 **100**。
