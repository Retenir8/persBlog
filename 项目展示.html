技术栈

项目采用 Next.js 全栈模式，前端、API、认证、数据库访问和 AI 服务都在同一个工程内组织。

Next.js 16 + App Router

负责页面路由、布局、服务端接口和整体应用结构。

React 19 + TypeScript

构建可维护的组件体系，并通过类型约束减少业务错误。

Tailwind CSS 4

支撑响应式布局、主题视觉和柔和的卡片式界面。

Prisma 5 + PostgreSQL

承载用户、文章、评论、好友、会话、消息和多媒体数据。

NextAuth v5 + bcryptjs

实现 Credentials 登录、JWT 会话和密码安全校验。

TipTap 富文本编辑器

为文章创作提供更接近内容平台的编辑体验。

小米 MiMo AI 接口

通过 OpenAI 兼容聊天接口驱动宠物问答与读后感。

NeteaseCloudMusicApi

支持音乐收藏、播放地址代理和个人主页音乐组件。

REST API 私信系统

以请求式同步完成会话、消息、已读、撤回、静音和置顶。

特色设计

项目最突出的部分是把 AI 陪伴与社交关系融入博客，而不是停留在单纯的内容管理。

AI 宠物「厨师猫」

全站常驻的陪伴角色，能互动、能养成、能聊天，也能阅读文章生成读后感。

拖拽定位与点击动画反馈 饱食度、心情、本地状态衰减 喂食面板与宠物食物配置 AI 问答与登录态文章读后感 服务端文章权限校验，避免越权读取

聊天交友系统

从关注关系延伸到互关好友、好友分组、站内私信和黑名单，形成轻社交闭环。

关注、取消关注与互关好友 好友分组参与文章可见性控制 一对一会话与消息列表 已读、未读角标、置顶、静音 2 分钟内撤回与文件消息上传

全部功能模块

当前功能已经覆盖个人博客、社交、内容扩展、后台管理和个性化主页等多个方向。

用户认证

注册、登录、JWT 会话、角色区分、资料编辑和个人设置。

博客文章

发布、编辑、删除、分类、标签、搜索、浏览量和我的文章。

互动评论

登录评论、游客评论、嵌套回复和评论管理。

点赞浏览

文章点赞状态、唯一点赞约束和阅读量统计。

文章权限

公开、互关好友、指定好友分组和不可见分组控制。

关注订阅

关注关系、互关好友、订阅信息流和用户主页关系操作。

好友分组

分组创建、成员维护，并与文章可见性联动。

站内私信

会话、消息、附件、已读、撤回、置顶、静音和未读数。

黑名单

提供社交安全基础，用于管理不希望互动的用户。

留言板

支持主页留言、游客留言和用户留言记录。

音乐空间

网易云歌曲和歌单收藏、播放代理、主页播放器组件。

书架记录

想读、在读、已读状态，支持封面、评分、备注和链接。

摄影相册

照片分类、图片展示、标题、排序和用户维度管理。

个性主页

个人资料、主页组件、数字时钟、心情日历、禅语和音乐组件。

管理后台

管理员页面和管理端内容接口，支持站点维护。

产品能力路径

从内容创作到社交互动，再到 AI 陪伴，项目已经形成完整的体验链路。

内容创作

用户用 TipTap 编辑文章，配置分类、标签和可见范围，发布到个人博客与信息流。

关系沉淀

读者关注作者，互关后成为好友，并可进入好友分组管理更精细的内容权限。

私信交流

好友之间通过站内私信交流，支持会话置顶、静音、已读、撤回和附件消息。

AI 陪伴

宠物全站陪伴用户，可被喂食、对话，也能基于文章内容生成读后感。

个人空间

音乐、书架、摄影、留言板和主页组件共同构成更完整的个人数字名片。

设计亮点总结

这些设计让项目区别于普通博客模板，更接近一个有生命感的个人社区空间。

AI 角色化

把 AI 包装为「宠物」而不是工具面板，让问答、读文章和状态养成都更自然。

关系驱动权限

好友分组不仅用于管理联系人，也参与文章访问控制，形成社交内容分发逻辑。

轻社交闭环

关注、互关、分组、私信、黑名单相互衔接，支撑从阅读到交流的转化。

多维个人表达

文章、音乐、书架、摄影、留言板、心情日历共同塑造完整的个人主页。
